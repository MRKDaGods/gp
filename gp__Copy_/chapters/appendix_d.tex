\section{Key Data Contracts}

\subsection{FrameInfo}

Produced by Stage~0; consumed by Stage~1.

\begin{verbatim}
{
  "frame_id":    0,             // 0-based integer
  "camera_id":   "S02_c009",
  "timestamp":   10.0,          // seconds from video start
  "frame_path":  "data/outputs/.../stage0/S02_c009/frame_000000.jpg",
  "width":       1920,
  "height":      1080
}
\end{verbatim}

\subsection{Tracklet}

Produced by Stage~1; consumed by Stages~2--4.

\begin{verbatim}
{
  "track_id":   1,
  "camera_id":  "S02_c009",
  "class_id":   2,
  "class_name": "car",
  "frames": [
    {
      "frame_id":   100,
      "timestamp":  10.0,
      "bbox":       [x1, y1, x2, y2],
      "confidence": 0.92
    }
  ]
}
\end{verbatim}

\subsection{GlobalTrajectory}

Produced by Stage~4; consumed by Stages~5--6 and the frontend.

\begin{verbatim}
{
  "global_id":   42,
  "class_name":  "car",
  "evidence": [
    {
      "camera_id": "S02_c009",
      "track_id":  1,
      "start_frame": 100,
      "end_frame":   250,
      "similarity":  0.87
    }
  ]
}
\end{verbatim}

\section{Configuration System}

All pipeline parameters are managed through OmegaConf YAML. The hierarchical config structure is:

\begin{verbatim}
configs/default.yaml           # Base config for all stages
configs/datasets/cityflowv2.yaml  # Dataset-specific overrides merged on top
CLI --override flags           # Applied last (highest priority)
\end{verbatim}

Config access follows the stage hierarchy:

\begin{verbatim}
cfg.stage1.tracker.algorithm                       # "deepocsort"
cfg.stage2.reid.pca_n_components                   # 384
cfg.stage4.association.graph.similarity_threshold  # 0.53
cfg.stage4.association.fic.regularisation          # 0.1
\end{verbatim}
